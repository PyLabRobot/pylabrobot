3.5 Device Overview 
The EVO _MCA controls three moving axis (x, y, z) and a plunger axis. An optional gripper axis (g) is also 
available. A CAN interface (HCAN) assures communications between the Device CU (DCU) with the master CU 
(TeCU). A local CAN bus (LCAN) communicates with a DCServo-2 module and a TeServo. The base address 
of the MCA device is 'W'.

3.6 Operation & Setup 
Preliminary actions have to be performed in order to operate the module. 
A. 
Installation of bootware 
B. Assure correct CAN-Bus termination (60 Ohm) 
C. Set address switch to the correct address 
D. Installation of firmware 
E. Hardware configuration 
F. 
Calibration 
Boot loading 
The boot itself cannot be loaded via the CAN interface. It has to be programmed on a dedicated flash 
programmer before the assembly. 
Firmware Download I Boot Sequence 
The firmware download is initiated with an SOF 
command if there is already a valid firmware running. The 
program now jumps into the boot mode. 
The boot program supports the following commands: 
• 
'A' followed by the hex file name checks if the 
first three characters of the hex file name does 
correspond with the three characters <'xxx'>. 
This protects the module from loading a firmware 
that belongs to another device. 
• 
'S' initiates the firmware download process. It 
applies the program voltage for the FLASH 
memory and copies the download procedure into 
the RAM memory range. 
• 
'E' erases the FLASH application page. 
• 
'H' followed by a byte stream writes the stream 
directly into the FLASH memory. An 'EOF' string 
indicates the end of the hex file. A redundancy 
checksum is calculated and compared with a 
desired value. The program jumps into the 
firmware page and executes the downloaded 
code. In case of a deviation between checksums, 
the program remains in the boot mode.

Z - Axis Overview 
The Z-Axis initializes to the upper Hard Stop position. It moves to the lnitOffset position after successful 
initialization. The positions are related to the tip bottom. The zero-position is equal to the worktable surface. 
Positions below the worktable are achieved with negative Z-positions. Each time the length of the tips gets 
changed, the Range and the TraversePath needs to be adopted accordingly. A solenoid controlled z-brake 
holds the head in position while power-off. The Z-brake is released automatically with any z-positioning or 
initialization command.
