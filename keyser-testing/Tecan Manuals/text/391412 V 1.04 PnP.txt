4.1 POSITIONING, MOVE AND ACTION COMMANDS: 
4.1.1 PIF 
FAKE INITIALIZATION  X/Y/Z/R/G - AXIS:                
The PIF command  initializes all axes. However, no physical initialization is executed. The current 
position of each axis are still valid. If power is disconnected and turned on, using the PIF command 
the axis  positions are not defined! CAUTION: Incorrect usage of this command may damage the 
instrument. 
EXAMPLE: #W2PIF 
Fake initialization of the P&P. 
RESPONSE: none 
GENERATED ERRORS: none            
4.1.2 PIA 
POSITION INITIALIZATION  X/Y/Z/R/G - AXES: 
This command initializes the P&P X/Y/Z/Rotator and Gripper axis. No movement command for the 
arm will be accepted before the PIA command is executed. When the PIA command is entered, all 
axis move to their initialization positions with a default init speed. After the init position of an axis 
has been detected, it moves the defined offset steps in the opposite direction.  
EXAMPLE: #W2PIA 
Initialization of all arm axis.     
4.1.3 PIX 
InitSpeed* 
POSITION INITIALIZATION  X - AXIS: 
This function re-initializes only the X-axis. An override motor speed for axis initialization may be 
placed at the end of the command. A command without any parameter uses the default speed to 
initialize. 
InitSpeed: 
speed in 0.1 mm/s  [5...1500]  (default = 205) 
EXAMPLE: #W2PIX250 
Initialization of the P&P X-axis with an init speed of 25 mm/s. 
RESPONSE: none 
GENERATED ERRORS: (1) init failed, (3) invalid operand, (5) device not implemented

4.1.4 PIY 
InitSpeed* 
POSITION INITIALIZATION  Y - AXIS:         
This function reinitializes only the Y-axis on the P&P. An override motor speed for axis initialization 
may be placed at the end of the command. A command without any parameter uses the default 
speed for initialization. 
InitSpeed: 
speed in 0.1 mm/s  [5...1500]  (default = 505)

Initialization of the P&P Y- axis with an init speed of 10 mm/s. 
RESPONSE: none

4.1.5 PIZ 
InitSpeed* 
POSITION INITIALIZATION  Z - AXIS:  
This function reinitializes only the Z-axis on the P&P. An override motor speed for axis initialization 
may be placed at the end of the command. A command without any parameters uses the default 
speed and all tips to initialize. 
InitSpeed: 
speed in 0.1 mm/s  [5...1500]  (default = 296) 
EXAMPLE: #W2PIZ170 
Initialization of the P&P Z-axis with an init speed of 17 mm/s. 
RESPONSE: none 
GENERATED ERRORS: (1) init failed, (3) invalid operand, (5) device not implemented 
4.1.6 PIR  
POSITION INITIALIZATION ALL DEPENDENT AXES:  
This function initializes all dependent axis of the P&P. First the Z-axis moves up, then the Y-axis 
moves to a save position, the rotator and the gripper are initialized afterwards. Finally the Z-axis is 
initialized again and moves to it’s offset position. It uses the default speed for initialization. 
EXAMPLE: #W2PIR 
Initialization of the P&P Z-axis, rotator and gripper. 
RESPONSE: none 
GENERATED ERRORS: (1) init failed, (5) device not implemented

4.1.31 AGR GripSearch*, GripOpen*  
GRIPP TUBE: 
This command grips with the gripper drive at the current X, Y, Z position. The gripper searches for 
a tube to the absolute GripSearch position. If there is no tube found, it re-opens to the absolute 
GripOpen position. An error message (tube not fetched) is sent back.  
GripSearch: 
[MinSpace..MaxSpace]  
 
GripOpen: 
[MinSpace..MaxSpace]

PICK TUBE:
